# GitHub Comment 输出模板

> SKILL.md Stage 4 发布到 PR 时使用此模板。

---

## 模板

```markdown
## Architecture Review: {PR Title}

### 变更概览
- **提交**: {N} 个提交（{commit_types}）
- **文件**: {N} 个文件（+{additions} / -{deletions}）
- **分类**: Code {n} | SQL {n} | Config {n} | Docs {n} | Other {n}
- **分支状态**: {同步/落后} {base_branch} {N} 个提交

### 评审结果

| # | 检查项 | 结果 | 说明 |
|---|--------|------|------|
| F1 | 分支同步 | ℹ️ | {同步/落后 N 提交} |
| F2 | 变更概览 | ℹ️ | {统计摘要} |
| F3 | Commit 规范 | ✅/❌ | {说明} |
| {D*} | {动态检查项名称} | ✅/⚠️/❌ | {说明} |

### 发现的问题

#### Critical
{阻塞合并的问题，如无则写"无"}

#### Important
{应修复但不阻塞的问题，如无则写"无"}

#### Suggestion
{改进建议，如无则写"无"}

### 建议
{具体行动建议，例如：}
{- 建议运行 `/mj-sys-doc-review` 检查文档质量}
{- 建议运行 `/mj-sys-git-check-merge` 确认技术合并就绪}

---
*Generated by mj-sys-git-review-pr skill*
```

---

## 结果图标说明

| 图标 | 含义 | 使用场景 |
|------|------|----------|
| ✅ | 通过 | 检查项完全符合规范 |
| ❌ | 未通过 | 检查项不符合规范（Critical 或 Important） |
| ⚠️ | 需人工判定 | 自动检查无法确定，需要人工确认 |
| ℹ️ | 信息展示 | 仅提供信息，不做合规判定（F1, F2） |
